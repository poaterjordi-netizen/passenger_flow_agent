# 使用指南

## 适合哪些用户

- 运营人员：查询、比较、异常候选、趋势和活动场景分析；
- 数据分析人员：检查 QueryIR、工具结果、证据和审计记录；
- 管理者与甲方：查看系统边界、验证状态、风险与生产缺口；
- 开发人员：复用 API、工具注册表和统一结果契约扩展能力。

## Web 页面

### 运营总览

查看合成指标卡、站点排行和确定性图表。页面用于验证信息结构，不代表实时生产客流。

### 智能分析

输入自然语言问题。系统返回完整 `RunRecord`，包括：

- 原始问题和结构化 `IntentEnvelope`；
- 受约束 `TaskPlan`；
- 每个工具的状态、参数摘要和结果；
- `EvidencePacket` 与证据编号；
- 最终回答、限制和建议；
- `VerificationReport`；
- Provider、模型、调用次数、token 可用性和耗时；
- 状态机事件与人工反馈入口。

若页面显示“未核验（禁止采纳）”，不得把结果当作正常建议继续使用。

### 受约束查询

适合熟悉指标口径的用户直接填写指标、维度、时间和过滤条件。请求仍要经过 QueryIR schema、指标目录和参数化查询模板。

### 基线预测

指定日预测复制参考日站点流入/流出模式，并保留方案元数据。它是可解释基线，不是已经验证准确率的生产预测模型。

## 推荐提问方式

| 目标 | 示例 | 预期工具链 |
| --- | --- | --- |
| 查询排序 | 查询各站进站客流并排序 | `query_metric → rank_stations` |
| 线路比较 | 比较 1 号线和 2 号线进站客流 | 并行查询 → `compare_groups` |
| 相关分析 | 客流变化与正点率是否相关 | 查询 → 运营指标 → 相关计算 |
| 活动预测 | 奥体中心周六有 4 万人演唱会，预测散场客流 | 相似日 → 基线 → 活动规则 → 对比 → SOP |
| 原因诊断 | 为什么某站今天客流下降 | 查询 → 运营指标 → 候选原因与证据缺口 |

当前合成目录不支持的线路、站点、时间或指标会触发澄清或拒绝，而不是静默扩大到全网。

## 如何判断结果能否采用

依次检查：

1. `status` 是否为 `completed`；
2. 所需工具是否成功；
3. 证据卡是否覆盖关键数字；
4. `verification.valid` 是否为 `true`；
5. 页面是否说明合成数据、预测基线或因果限制；
6. 是否涉及通知、处置、真实数据或生产动作；若涉及，必须由授权人员确认。

## 当前边界

- 默认数据是合成数据；
- 模型候选意图和计划必须与确定性 protected baseline 关键字段严格一致，因此当前模型规划增益有意受限；
- 相关分析只能说明相关或候选原因，不能自动证明因果；
- 活动、GIS、公交换乘、实时信号和 SOP 路径目前使用合成夹具；
- 不发送通知、不安装定时任务、不自动采取运营动作。
